# Configuration Overview

Every config follows the same principle: **modular, commented, and sourced from matugen colors where possible.**

## Hyprland

**Location:** `~/.config/hypr/`

The Hyprland config is written in Lua for readability and modularity.

```
hypr/
├── hyprland.lua            # Entry point, loads modules
├── colors.lua              # Matugen-generated color palette
├── hyprlock.conf           # Lock screen config
├── hyprlock-colors.conf    # Matugen colors for hyprlock
├── hypridle.conf           # Idle management
└── modules/
    ├── monitors.lua        # Monitor setup
    ├── binds.lua           # Keybindings
    ├── autostart.lua       # Startup applications
    ├── env.lua             # Environment variables
    ├── input.lua           # Input devices (keyboard, mouse, touchpad)
    ├── looknfeel.lua       # Visual settings (gaps, borders, blur, shadows)
    ├── layout.lua          # Window layout settings
    ├── misc.lua            # Miscellaneous settings
    └── windowrules.lua     # Window rules (floating, blur, opacity)
```

### Colors

`colors.lua` is auto-generated by matugen. Do not edit it directly. All modules import it to use the current wallpaper palette.

```lua
-- Example: how looknfeel.lua uses colors
local colors = require("colors")
hl.config({
    general = {
        col = {
            active_border = colors.primary,
            inactive_border = colors.outline_variant,
        },
    },
    decoration = {
        shadow = { color = colors.shadow },
    },
})
```

## Matugen

**Location:** `~/.config/matugen/`

Matugen generates colors from your wallpaper and outputs them to templates.

```
matugen/
├── config.toml             # Template definitions
├── colors.css              # CSS variables
└── templates/
    ├── hyprland-colors.lua
    ├── hyprlock-colors.conf
    ├── waybar-colors.css
    ├── wlogout-colors.css
    ├── rofi-colors.rasi
    ├── gtk-css.css
    ├── gtk4-css.css
    ├── qt5ct-colors.conf
    ├── qt6ct-colors.conf
    ├── cava-config
    ├── nvim-matugen.lua
    ├── firefox-userChrome.css
    ├── fastfetch-config.jsonc
    ├── swaync-variables.css
    ├── kitty-theme.conf
    └── ghostty-theme.conf
```

When you change your wallpaper via waypaper, `matugen-apply-colors` runs and regenerates every template file, then triggers app reloads via post_hooks (e.g., `pkill -SIGUSR2 waybar`, `hyprctl reload`).

## GTK

**Location:** `~/.config/gtk-3.0/`, `~/.config/gtk-4.0/`

Both GTK3 and GTK4 use `gtk.css` generated by matugen with `@define-color` variables.

`settings.ini` sets:
- `gtk-theme-name=Adwaita` (base theme)
- `gtk-application-prefer-dark-theme=1` (dark mode)
- Custom cursor theme: `Bibata-Modern-Ice`

## Qt

**Location:** `~/.config/qt5ct/`, `~/.config/qt6ct/`

Both Qt5 and Qt6 use `Fusion` style with a custom matugen color scheme.

The environment variable `QT_QPA_PLATFORMTHEME=qt6ct` is set in Hyprland's env module, ensuring all Qt apps pick up the theme.

The color scheme defines all 21 Qt color roles for active, disabled, and inactive states, mapped to matugen palette colors.

## Firefox

**Location:** `~/.mozilla/firefox/chrome/`

Firefox uses `userChrome.css` generated by matugen to style the browser UI:
- Tab bar, URL bar, sidebar, bookmarks toolbar
- Context menus, panels, find bar
- All styled with CSS variables from matugen

`user.js` enables `toolkit.legacyUserProfileCustomizations.stylesheets` and GTK dark theme integration.

## Brave / Chromium

Uses the system GTK theme for its UI. Configured via preferences for:
- Dark mode
- GTK theme integration
- System title bar

Flags file at `~/.config/brave-origin-nightly/flags.conf` passes `--force-dark-mode` on launch.

# Troubleshooting

## Hyprlock not working

**Symptom:** Lock screen crashes or doesn't appear.

**Check:** Make sure `hyprpolkitagent` is running.

```bash
systemctl --user status hyprpolkitagent
```

If it's not running, start it:
```bash
systemctl --user enable --now hyprpolkitagent
```

It's also launched in `hypr/modules/autostart.lua`:
```lua
hl.exec_cmd("systemctl --user start hyprpolkitagent")
```

**Also check:** The PAM config at `/etc/pam.d/hyprlock` should include `auth include login`.

## Hyprlock colors not applying

**Symptom:** Lock screen shows default colors, not matugen palette.

**Check:** The `hyprlock.conf` must have a `source` line pointing to the colors file:

```conf
source = /home/matrix/.config/hypr/hyprlock-colors.conf
```

Then use `$variable` in your config for colors.

## Wlogout slow to appear

**Symptom:** 3-5 second delay before the logout menu shows.

**Cause:** The blurred wallpaper image was too large.

**Fix:** The `generate-wlogout-blur` script now produces a 1920×1080 JPG at 85% quality instead of a 5K PNG. If the blur image is missing, regenerate it:

```bash
/home/matrix/.local/bin/generate-wlogout-blur
```

The blur is pre-generated by matugen's post_hook, so it should exist before wlogout launches.

## Wlogout buttons show emoji instead of icons

**Symptom:** Buttons display text like "🔒 Lock" instead of SVG icons.

**Check:** The layout file must use labels matching CSS selectors:

```json
{
    "label": "lock",
    "action": "hyprlock --immediate-render",
    "text": "Lock",
    "keybind": "l"
}
```

And `style.css` must have `#lock { background-image: url(...) }` selectors.

## GTK apps not themed

**Symptom:** GTK apps show light theme or default Adwaita colors.

**Check:** `~/.config/gtk-3.0/settings.ini` should have:
```ini
gtk-application-prefer-dark-theme=1
```

The `gtk.css` file in `~/.config/gtk-3.0/` is loaded automatically by GTK3 apps. If it exists with matugen colors, apps should pick it up.

If not, run `nwg-look` to apply the theme settings.

## Qt apps not themed

**Symptom:** Qt apps (Dolphin, Okular, etc.) show light theme.

**Check 1:** The environment variable must be set:
```bash
echo $QT_QPA_PLATFORMTHEME
# Should print: qt6ct
```

If not, it's set in `hypr/modules/env.lua`:
```lua
hl.env("QT_QPA_PLATFORMTHEME", "qt6ct")
```

**Check 2:** Open `qt6ct` GUI and verify:
- Style: `Fusion`
- Color scheme: `matugen` (custom)
- Icon theme: `Adwaita`

**Check 3:** The color scheme file at `~/.config/qt6ct/colors/matugen.conf` should have all 21 color roles defined for active, disabled, and inactive states.

## Firefox not themed

**Symptom:** Firefox shows default light theme.

**Check 1:** Make sure `userChrome.css` exists at `~/.mozilla/firefox/chrome/userChrome.css`.

**Check 2:** Firefox requires `toolkit.legacyUserProfileCustomizations.stylesheets = true` in `about:config`. The `user.js` should set this automatically, but you can verify in `about:config`.

**Check 3:** The `user.js` must be in your active profile directory (e.g., `~/.mozilla/firefox/xxxxxxxx.default-release/user.js`).

## Colors not updating after wallpaper change

**Symptom:** Changing wallpaper doesn't update app colors.

**Check 1:** Did `matugen-apply-colors` run? Check the wallpaper cache:
```bash
cat ~/.cache/current-wallpaper
```

**Check 2:** Run it manually:
```bash
matugen-apply-colors ~/Pictures/Wallpapers/your-image.jpg
```

**Check 3:** Check that templates generated correctly:
```bash
cat ~/.config/nvim/lua/generated/matugen.lua  # should have hex colors
cat ~/.config/waybar/tokens/colors.css         # should have CSS variables
```

**Check 4:** Some apps need a restart to pick up new colors (Neovim, Firefox, Qt apps). Waybar auto-reloads via SIGUSR2.

## Hyprland not starting

**Symptom:** Black screen or fallback to TTY after login.

**Check:** Select "Hyprland" (not "Hyprland (fallback)" or "Hyprland (without polkit)") from your display manager.

**Check logs:**
```bash
journalctl --user -f -n 50
```

Common issues:
- Missing `hyprland.lua` in `~/.config/hypr/`
- Broken Lua syntax in config files
- Missing environment variables

## Cava not showing audio

**Symptom:** Cava visualizer shows flat bars or nothing.

**Check:** Cava uses PipeWire as input method. Verify PipeWire is running:
```bash
pactl info
```

If you're using a different audio system, change the `[input]` section in `~/.config/cava/config`:
```ini
[input]
method = pulse  # or pipewire, sndio, etc.
source = auto
```

## General debugging

**Check config syntax:**
```bash
hyprctl configinfo  # validates Hyprland config
```

**Check for errors in real-time:**
```bash
journalctl --user -f -o cat | grep -i error
```

**Reset a specific config** by removing the symlink and letting the installer recreate it:
```bash
rm ~/.config/hypr
cd ~/hyprland_dotfiles
./install.sh
```
